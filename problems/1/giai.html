LÀM SẠCH DỮ LIỆU (UNIQUE)

Kỳ thi: HSG Tin Học Điểm: 100 Thời gian: 1.0s

📖 Đề bài

Trong một thư viện số hiện đại, một robot có nhiệm vụ thu thập các mã số định danh của các cuốn sách quý. Do quá trình quét dữ liệu diễn ra liên tục, hệ thống vô tình ghi nhận nhiều mã số bị lặp lại. Các mã số này vốn đã được robot sắp xếp theo thứ tự không giảm (tăng dần).

Để tiết kiệm bộ nhớ lưu trữ, người quản trị yêu cầu bạn hãy loại bỏ tất cả các mã số trùng lặp, chỉ giữ lại duy nhất một đại diện cho mỗi loại mã. Hãy xác định xem sau khi "làm sạch" dữ liệu, thư viện còn lại bao nhiêu mã số khác nhau?

Dữ liệu vào (Input)

  • Dòng đầu tiên chứa số nguyên dương $n$ là số lượng phần tử của dãy ban đầu ($1 \le n \le 10^6$).
  • Dòng thứ hai chứa $n$ số nguyên $a_1, a_2, ..., a_n$ ($-10^9 \le a_i \le 10^9$) đã được sắp xếp tăng dần, các số cách nhau bởi dấu cách.

Dữ liệu ra (Output)

  • Ghi một số nguyên duy nhất là số lượng các phần tử khác nhau trong dãy.

Ví dụ (Sample Test)

UNIQUE.INP UNIQUE.OUT
10 0 0 1 1 1 2 2 3 3 4 5

🧠 Phân tích Logic & Thuật toán

Dạng bài: Xử lý mảng (Array), Cấu trúc dữ liệu (Set), Kỹ thuật Hai con trỏ / Duyệt tuyến tính (Linear Scan).

Phân tích tính chất: Bài toán yêu cầu đếm số lượng phần tử phân biệt. Chúng ta có thể tiếp cận theo 3 hướng: sử dụng mảng phụ để lưu trữ (chậm), sử dụng cấu trúc dữ liệu Set (nhanh nhưng tốn bộ nhớ), hoặc tận dụng tính chất đã sắp xếp tăng dần của dữ liệu đầu vào để giải quyết tại chỗ (nhanh và tối ưu bộ nhớ nhất).

Lưu ý giới hạn (Constraints Warning)

  • Giới hạn thời gian: $N \le 10^6$. Giải thuật có độ phức tạp $O(N^2)$ sẽ thực hiện khoảng $10^{12}$ phép toán và chắc chắn bị Time Limit Exceeded (TLE). Phải thiết kế giải thuật $O(N)$.
  • Kiểu dữ liệu: Giá trị $A_i$ có thể lên tới $10^9$. Python tự động quản lý số lớn nên không lo tràn số (Integer Overflow). Trong C++/Java cần dùng kiểu `long long` nếu có phép tính tổng, tuy nhiên ở đây chỉ so sánh nên `int` 32-bit là đủ.
  • Bộ nhớ: Cần cẩn trọng với các cấu trúc dữ liệu sinh thêm như Set khi $N = 10^6$, nó có thể tốn khá nhiều RAM (Memory Limit).

⚙️ Hướng dẫn giải chi tiết

Cách 1: Thuật toán Ngây thơ (Dùng Mảng) - Cho 30% số điểm

Ý tưởng: Khởi tạo một mảng result rỗng. Duyệt qua từng phần tử của mảng ban đầu, nếu phần tử đó chưa xuất hiện trong result thì thêm vào. Thuật toán này bỏ qua tính chất "đã sắp xếp". Do phép kiểm tra tồn tại trong mảng mất $O(K)$, tổng thời gian là $O(N^2)$.

Khởi tạo mảng result = []
Với mỗi x trong a:
    Nếu x không nằm trong result:
        Thêm x vào result
In ra độ dài result 
# Cách 1: Bỏ qua tính chất sắp xếp - O(N^2) - Sẽ bị TLE
n = int(input())
a = list(map(int, input().split()))

result = []
for x in a:
    if x not in result: # Tìm kiếm tuyến tính tốn O(K)
        result.append(x)

print(len(result))

Cách 2: Sử dụng Cấu trúc dữ liệu Set (Tập hợp) - Cho 60-100% số điểm

Ý tưởng: Set trong Python được cài đặt bằng Hash Table, giúp thao tác thêm và kiểm tra tồn tại chỉ mất $O(1)$ trung bình. Ta chỉ cần ném toàn bộ mảng vào `Set`, tự động các phần tử trùng lặp sẽ bị loại bỏ. Mặc dù thời gian là $O(N)$, nhưng việc cấp phát Hash Table cho $10^6$ phần tử tốn nhiều không gian bộ nhớ phụ $O(N)$.

Khởi tạo tập hợp my_set = tập rỗng
Với mỗi x trong a:
    Thêm x vào my_set (Set tự động loại bỏ trùng lặp)
In ra kích thước của my_set 
# Cách 2: Dùng Set - O(N) Thời gian, O(N) Không gian
import sys

input_data = sys.stdin.read().split()
if not input_data:
    sys.exit()

n = int(input_data[0])
a = [int(x) for x in input_data[1:n+1]]

# Chuyển list thành set để tự động lọc trùng
unique_elements = set(a)

print(len(unique_elements))

Cách 3: Linear Scan (Quét tuyến tính tận dụng mảng đã sắp xếp) - Cho 100% số điểm

Ý tưởng: Lợi dụng tính chất mảng đã được sắp xếp, các phần tử giống nhau chắc chắn đứng cạnh nhau. Ta chỉ cần duyệt mảng một lần, so sánh phần tử hiện tại a[i] với phần tử liền trước a[i-1]. Nếu khác nhau, ta đếm thêm 1. Giải pháp này không cần cấu trúc dữ liệu phụ, đạt độ phức tạp không gian lý tưởng $O(1)$.

Nếu n == 0 thì in 0 và kết thúc.
Khởi tạo count = 1 (Luôn tính phần tử đầu tiên)
Chạy i từ 1 đến n - 1:
    Nếu a[i] != a[i-1]:
        count = count + 1
In ra count 
# Cách 3: Linear Scan - O(N) Thời gian, O(1) Không gian
import sys

def solve():
    input_data = sys.stdin.read().split()
    if not input_data:
        return
        
    n = int(input_data[0])
    if n == 0:
        print(0)
        return
        
    a = [int(x) for x in input_data[1:n+1]]
    
    count = 1
    for i in range(1, n):
        if a[i] != a[i-1]:
            count += 1
            
    print(count)

if __name__ == '__main__':
    solve()
