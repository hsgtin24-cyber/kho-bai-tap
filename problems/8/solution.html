ĐẾM DÃY CON CÓ TỔNG CHIA HẾT CHO K

Chuyên đề: Mảng cộng dồn (Prefix Sum) & Đồng dư thức (Modular Arithmetic)

Thử Thách | Độ khó: Medium-Hard

Phần 0 Đề Bài Chi Tiết

Trong một lễ hội toán học, ban tổ chức đưa ra một dãy các hộp quà đánh số từ 1 đến $n$, mỗi hộp quà chứa một thẻ số nguyên. Thử thách dành cho các bạn học sinh là tìm ra những đoạn quà liên tiếp sao cho tổng các giá trị ghi trên thẻ trong đoạn đó là một số chia hết cho số may mắn $k$ của năm nay.

Yêu cầu:

Cho dãy số nguyên $A$ gồm $n$ phần tử và số nguyên $k$ ($k \neq 0$). Hãy đếm số lượng dãy con liên tiếp trong dãy $A$ sao cho tổng các phần tử của dãy con đó chia hết cho $k$.

Dữ liệu vào (Input)

  • Dòng 1: Ghi hai số nguyên $n$ và $k$ ($1 \le n \le 10^5, 2 \le |k| \le 10^5$).
  • Dòng 2: Ghi $n$ số nguyên của dãy $A$ (giá trị tuyệt đối của mỗi số không quá $10^9$).

Dữ liệu ra (Output)

Một số nguyên duy nhất là số lượng dãy con liên tiếp tìm được.

Ví dụ (Sample Test)

DIVISIBLE.INP DIVISIBLE.OUT GIẢI THÍCH
5 3 4 5 0 -2 6 6 Các dãy con có tổng chia hết cho 3 là:
1. [4, 5] (Tổng = 9)
2. [4, 5, 0] (Tổng = 9)
3. [0] (Tổng = 0)
4. [5, 0, -2] (Tổng = 3)
5. [5, 0, -2, 6] (Tổng = 9)
6. [6] (Tổng = 6)

Phần 1 Phân Tích Toán Học & Logic

Dạng bài: Đếm dãy con dựa trên Đồng dư thức (Subarray Sums Divisible by K).

Cơ sở toán học: Gọi $P[i]$ là tổng cộng dồn (Prefix Sum) từ đầu mảng đến phần tử thứ $i$. Tổng của dãy con từ $i$ đến $j$ là: $$\sum_{m=i}^{j} A[m] = P[j] - P[i-1]$$

Đề bài yêu cầu tổng dãy con chia hết cho $k$, tức là:

$(P[j] - P[i-1]) \equiv 0 \pmod k \iff P[j] \equiv P[i-1] \pmod k$

Kết luận cực kỳ quan trọng: Hai vị trí có cùng số dư (khi chia cho $k$) sẽ tạo thành một dãy con có tổng chia hết cho $k$. Vậy thay vì lặp qua từng dãy con, ta chỉ cần đếm tần suất xuất hiện của các số dư từ mảng Prefix Sum, sau đó lấy tổ hợp chập 2, hoặc cộng dồn trực tiếp khi lặp.

CẢNH BÁO: TOÁN TỬ MODULO VỚI SỐ ÂM

  • Trong C++/Java/C#, toán tử % có thể trả về số âm (vd: -2 % 3 = -2). Trong toán học đồng dư, số dư phải $\ge 0$.
  • Cách xử lý chuẩn: rem = ((current_sum % k) + k) % k. Phép toán này đảm bảo số dư luôn nằm trong khoảng $[0, k-1]$.
  • Python thì % tự động xử lý trả về số dương nên current_sum % k là đủ, nhưng cẩn thận khi chuyển ngữ qua C++.

Phần 2 Phân Tích Cách Giải

Cách 1: Vét cạn (Brute Force) - Đáp ứng Subtask N ≤ 1000

Duyệt mọi cặp chỉ số đầu $i$ và cuối $j$, tính tổng đoạn và kiểm tra chia hết cho $k$.

Khởi tạo ans = 0
Với mỗi i từ 0 đến n-1:
  sum = 0
  Với mỗi j từ i đến n-1:
    sum += A[j]
    Nếu sum % k == 0 thì ans += 1
Trả về ans 
def solve_brute_force(A, k):
    n = len(A)
    ans = 0
    for i in range(n):
        current_sum = 0
        for j in range(i, n):
            current_sum += A[j]
            if current_sum % k == 0:
                ans += 1
    return ans

Cách 2: Prefix Sum + Hash Map (Tối ưu) - AC 100% Điểm

Sử dụng mảng hoặc Dictionary (Bảng băm) để lưu tần suất các số dư. Đừng quên khởi tạo freq[0] = 1 vì nếu bản thân current_sum chia hết cho $k$, nó tạo thành một dãy con hợp lệ tính từ đầu mảng.

Khởi tạo mảng freq cỡ k với tất cả giá trị là 0
freq[0] = 1
Khởi tạo current_sum = 0, count = 0
Duyệt x trong mảng A:
  current_sum += x
  rem = ((current_sum % k) + k) % k
  count += freq[rem]
  freq[rem] += 1
Trả về count 
def solve_optimal(A, k):
    # Dùng Dictionary đếm số dư
    freq = {0: 1} 
    current_sum = 0
    count = 0
    
    for x in A:
        current_sum += x
        # Đảm bảo số dư luôn dương
        rem = ((current_sum % k) + k) % k
        
        if rem in freq:
            count += freq[rem]
            freq[rem] += 1
        else:
            freq[rem] = 1
            
    return count
